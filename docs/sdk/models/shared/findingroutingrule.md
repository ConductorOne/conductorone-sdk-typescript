# FindingRoutingRule

FindingRoutingRule configures what happens when a finding is detected.
 Rules are evaluated in two passes: tenant-wide rules first (global override),
 then app-scoped rules (local customization). First match wins within each pass.

 Matching uses CEL expressions evaluated against the Finding object.
 Example conditions:
   finding.severity == FindingSeverity.CRITICAL
   finding.custom_tags["env"] == "production"
   finding.risk_score > 75 && finding.recurrence_count >= 3

## Example Usage

```typescript
import { FindingRoutingRule } from "conductorone-sdk-typescript/sdk/models/shared";

let value: FindingRoutingRule = {};
```
